# 🏆 DLLM: 20-Minute Knowledge Access Network

## 🎯 **專案概述**

**DLLM (Distributed Local Library Module)** 是一個開源的去中心化圖書共享平台，旨在建立一個去中心化的本地圖書共享社區。本專案將 DLLM 系統與 **20-Minute Neighbourhood Plan** 挑戰目標完美結合，創造一個真正由下而上、「無大台」的知識共享網絡。

### **如何達到 20-Minute Neighbourhood Plan 挑戰目標**

#### **🎯 核心契合點**
- **社區賦權**: DLLM 讓每個居民成為知識分享者，實現真正的社區自主
- **社區連結**: 透過書籍流通建立人際網絡，促進社區互動和凝聚力
- **資源可及性**: 確保居民能在20分鐘內獲得學習資源和知識

#### **🏘️ 20分鐘社區實現**
- **智能交收點規劃**: 基於20分鐘可及性原則規劃社區交收點
- **位置跟隨邏輯**: 書籍位置自動跟隨擁有者/持有人，確保資源可及性
- **交通時間優化**: 結合步行、騎車、公車等不同交通方式計算實際可及性
- **連鎖借閱網絡**: 利用「書主—持有人—借書人」多點傳遞模式擴大資源覆蓋

#### **🌍 社會影響**
- **政策支持**: 直接支持維多利亞州20分鐘社區政策目標
- **包容性**: 特別關注弱勢群體的教育資源需求
- **可擴展性**: 解決方案可應用於其他地區和社區

## 📋 **專案描述**

### **問題定義**
傳統圖書館面臨以下挑戰：
- **空間有限**: 實體圖書館無法容納所有書籍
- **程序繁複**: 借閱流程複雜，缺乏靈活性
- **資源不均**: 某些社區缺乏足夠的學習資源
- **20分鐘挑戰**: 居民無法在合理時間內獲得所需資源

### **解決方案**
DLLM 採用創新的「chain 借」（連鎖借閱）概念：
- **去中心化架構**: 每個居民都可以成為知識分享者
- **智能位置管理**: 書籍位置自動跟隨，確保資源可及性
- **社區交收點**: 結合現有設施建立智能交收網絡
- **連鎖傳遞**: 書籍透過多點傳遞模式在社區中流通

### **核心功能**
1. **書籍註冊和管理**: 支持多語言、多狀態的書籍信息管理
2. **智能借閱系統**: 三方協調的借閱申請和確認流程
3. **位置跟隨機制**: 自動更新書籍位置和持有人信息
4. **社區交收網絡**: 基於20分鐘可及性的交收點規劃
5. **多語言支持**: 內建繁體中文和英文支持

## 🏗️ **技術架構**

### **系統架構**
```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   前端客戶端     │    │   GraphQL API   │    │   數據存儲      │
│  (React + Vite) │◄──►│  (Apollo Server)│◄──►│  (Firestore)    │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         │                       │                       │
         │                       │                       │
         ▼                       ▼                       ▼
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   地圖服務      │    │   圖片處理      │    │   認證服務      │
│ (OpenStreetMap) │    │   (Sharp)       │    │ (Firebase Auth) │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

### **技術棧**
- **前端**: React 19 + Vite + Material-UI + TypeScript
- **後端**: Firebase Functions + Apollo GraphQL Server
- **數據庫**: Firebase Firestore (NoSQL)
- **地圖服務**: OpenStreetMap + Leaflet.js
- **圖片處理**: Sharp + Google Cloud Storage
- **認證**: Firebase Auth
- **國際化**: react-i18next

### **關鍵技術特點**
- **GraphQL 優先**: Schema 優先開發，自動化類型生成
- **地理空間查詢**: 使用 geofire-common 進行位置相關查詢
- **混合渲染**: 用戶使用 CSR，社交媒體機器人使用 SSR
- **機器人偵測**: 智能識別和處理不同類型的訪問者

## 🚀 **安裝說明**

### **環境要求**
- Node.js 18+ 
- npm 或 yarn
- Firebase CLI
- Git

### **快速開始**

#### **1. 克隆專案**
```bash
git clone [repository-url]
cd dllm-library
```

#### **2. 安裝依賴**
```bash
# 安裝客戶端依賴
cd client
npm install

# 安裝函數依賴
cd ../functions
npm install
```

#### **3. 配置 Firebase**
```bash
# 安裝 Firebase CLI
npm install -g firebase-tools

# 登入 Firebase
firebase login

# 初始化專案
firebase init
```

#### **4. 配置環境變量**
```bash
# 在 functions 目錄創建 .env 文件
cp .env.example .env
# 編輯 .env 文件，填入必要的配置信息
```

#### **5. 啟動開發環境**
```bash
# 啟動客戶端開發服務器
cd client
npm run dev

# 啟動 Firebase 模擬器
firebase emulators:start
```

### **開發指令**

#### **客戶端開發**
```bash
npm run dev          # 啟動開發服務器
npm run build        # 建置生產版本
npm test             # 執行測試
npm run codegen      # 生成 GraphQL 類型
```

#### **後端開發**
```bash
npm run dev          # 啟動本地開發服務器
npm run build        # 編譯 TypeScript
npm run start:inspect # 支援熱重載的開發模式
```

#### **部署**
```bash
# 部署後端函數
firebase deploy --only functions

# 部署客戶端
firebase deploy --only hosting
```

## 📚 **專案概述**

### **Epic 1: 用戶帳戶管理**
- **用戶註冊**: 簡化的註冊流程，支持郵箱驗證
- **用戶登入**: 安全的認證系統，支持會話管理
- **個人資料編輯**: 位置設置和地圖整合

### **Epic 2: 書籍上傳和放置**
- **書籍註冊**: 支持多語言、多狀態的書籍信息錄入
- **位置管理**: 智能的位置跟隨和交收點設置
- **狀態追蹤**: 實時狀態更新和變更歷史記錄

### **Epic 3: 書籍借閱**
- **搜索發現**: 全文搜索和地理位置篩選
- **借閱申請**: 三方協調的申請和確認流程
- **交收管理**: 靈活的交收方式和狀態追蹤

### **Epic 4: 借閱管理**
- **狀態追蹤**: 實時借閱狀態和歷史記錄
- **續借歸還**: 簡化的續借和歸還流程

## 📊 **數據使用說明**

### **開放數據整合**
- **VISTA 數據**: 分析居民移動模式和交通偏好
- **Vicmap 數據**: 精確的地理空間分析
- **學校位置**: 規劃教育資源可及性
- **公共交通**: 結合現有交通網絡

### **數據處理流程**
1. **數據收集**: 從多個開放數據源收集相關信息
2. **數據清洗**: 標準化和預處理原始數據
3. **數據整合**: 將不同來源的數據進行融合
4. **分析建模**: 建立20分鐘可及性分析模型
5. **可視化展示**: 創建直觀的數據儀表板

### **隱私保護**
- **地址隱私**: 用戶可選擇隱藏詳細地址信息
- **數據加密**: 敏感信息使用加密存儲
- **權限控制**: 基於角色的數據訪問控制
