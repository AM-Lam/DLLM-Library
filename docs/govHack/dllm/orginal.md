DLLM (Distributed Local Library Module) ：去中心化圖書館，打造本地知識共享社區
傳統圖書館一直係社會知識流通嘅核心，但同時都有唔少限制：
空間有限，唔可能收錄晒所有書。

書籍借還要經過中心化管理，程序繁複。

特殊書籍、罕有典籍，往往無緣被大眾接觸。

DLLM（Distributed Local Library Module） 嘅出現，正正係要 補足傳統圖書館嘅不足。DLLM 唔係要取代圖書館，而係用去中心化方式，將圖書館嘅概念延伸到社區每一個角落，令「書」可以更自由、更靈活咁流通。

去中心化的力量：一本書可以「chain 借」
DLLM 嘅理念好簡單：
書主（owner）、借書人（requester）、持有人（holder）都可以係不同人。

書籍唔需要一手交一手，可以經多個人傳遞（chain 借）。

書主可以選擇公開或者保留私隱，靈活度更高。

咁樣做，令一本書嘅生命週期變得更長，唔再局限於圖書館書架，而係真正流通到社區。就算係珍藏書、絕版書，喺 DLLM 嘅網絡入面，都有機會被更多人發現同閱讀。

來自真實輸入嘅測試：138 本書的故事
系統開發唔淨係紙上談兵。八月初，團隊五人親身去到 李偉才博士（YouTube「浩浩熵熵」主持人） 家中，將佢部分藏書輸入 DLLM。當日共輸入咗 138 本書，而且有意無意地「cosplay」唔同人嘅輸入風格，有人寫描述、有人冇寫，務求製造最混亂嘅數據，方便之後測試 UI 點樣處理。
過程中，我哋仲發現咗好多傳統圖書館未必會記錄嘅細節：例如「作者其實只係譯者」、或者某啲特殊典籍嘅私人筆記。呢啲真實案例，正正突顯咗 DLLM 可以比傳統系統捕捉到更多「在地化」同「人性化」嘅數據。
當然，問題亦即刻浮現：
輸入太多書，系統 load 得太慢，要加 pagination。

新書 category 排位唔理想。

入咗書冇得 edit 同 delete。

Free form category search 唔準確。

Transaction 記錄有時搵唔返。

雖然係 bug，但亦係好消息 —— 因為有真實數據測試，問題可以早早浮現，推動系統更快迭代。

超越圖書館：公開交收點與社區連結
DLLM 唔單止係個「技術項目」，而係一個 幫助社區連結嘅工具。
例如，我哋已經同一間 NGO 傾緊，佢哋原本俾家長交收玩具，而家可以加多一個功能：變成 書本交收點。咁樣，DLLM 就唔只係一個 app，而係真正走入社區，令書流通同人流通結合。
想像下：
你住喺將軍澳，但返工喺中環；DLLM 可以幫你揀出最方便嘅交收地點。

書唔再需要局限於「一館一地」，而係以「社區」為單位，形成鬆散但有力嘅網絡。

呢種模式，可以話係 去中心化圖書館最重要嘅價值 —— 幫助本地社區自己建立知識共享平台。

Open Source：真正做到「無大台」
DLLM 最大嘅特點，係以 開源（open source）軟件 形式發佈。
任何社區都可以 fork 同 host 自己版本。

唔同地區可以加返自己特色功能，例如社區活動、地道分類法、本地語言 tag。

避免單一平台壟斷，真正做到 「無大台」。

呢個 approach，令 DLLM 唔再只係一個中央團隊嘅產品，而係一個 社區自發、由下而上 嘅知識共享基建。每個地方，都可以有自己嘅 DLLM，既保持自主，又能透過 API / Federation 互相連結，兼容本地特色同全球交流。

技術進展：從 staging 到 production
開發過程方面，DLLM 團隊已經完成咗幾個 milestone：
自動生成縮圖：減少流量消耗，用戶 click 先放大。

User profile：可以睇到其他用戶，知道佢哋藏書。

Auto deploy with GitHub Action：由 staging 推去 production 完全自動化。

SSR 測試成功：準備未來更大規模嘅用戶流量。

雖然仲有功能未完善（例如 profile list 唔夠靚、未有 highlight rare 書嘅方法），但方向已經越來越清晰。

DLLM 未來展望
DLLM 並唔想做一個「取代圖書館」嘅產品，而係一個 補足圖書館不足嘅社區工具。佢可以：
幫助圖書館以外嘅書籍流通。

捕捉人性化、細緻化嘅書籍 metadata。

透過公開交收點，令社區成員互相連結。

利用開源模式，令各地社區都可以 host 自己版本，保持自主同特色，避免單一化。

而當 DLLM 慢慢流出嚟，社區開始試用，我哋相信呢個小小 experiment，可以成為一個由下而上嘅去中心化知識網絡。
