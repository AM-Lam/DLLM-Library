# 📚 DLLM 核心功能 - User Stories

## 🎯 **專案概述**

**系統名稱**: DLLM (Distributed Local Library Module)  
**核心概念**: 去中心化本地圖書共享社區  
**主要功能**: 書籍上傳/放置、借閱管理、社區圖書流通

## 🚀 **系統願景**

DLLM 是一個開源的去中心化圖書共享平台，旨在建立一個去中心化的本地圖書共享社區。它並非要取代傳統圖書館，而是要彌補其在空間、管理及藏書種類上的不足。

**核心概念**: 採用「chain 借」（連鎖借閱）概念，讓書籍透過「書主—持有人—借書人」的多點傳遞模式，在社區中更自由、靈活地流通。

**開發理念**: 強調透過真實數據測試來驅動開發，並積極與社區組織連結，設立實體交收點，將線上系統與線下人際網絡結合。

**開源精神**: 採用開源（Open Source）模式，讓任何社區都能建立自己的版本，避免單一平台壟斷，實現一個真正由下而上、「無大台」的知識共享網絡。

## 👥 **主要用戶角色 (User Personas)**

### **1. 書主 (Book Owners)**
- **年齡**: 25-70歲
- **技術水平**: 中等
- **動機**: 分享知識，延長書籍生命週期，讓珍貴或絕版書有更多機會在社區流通
- **痛點**: 擔心書籍遺失，缺乏借閱管理工具，需要保障私隱的靈活性

### **2. 借書人 (Book Borrowers/Requesters)**
- **年齡**: 18-65歲
- **技術水平**: 基礎到中等
- **動機**: 獲得學習資源，參與社區知識共享，解決傳統圖書館空間有限、程序繁複、無法觸及罕有書籍等問題
- **痛點**: 不知道社區內有什麼書籍可用

### **3. 書籍持有人 (Book Holders)**
- **年齡**: 20-60歲
- **技術水平**: 基礎到中等
- **動機**: 暫時保管書籍，協助社區流通，參與「書主—持有人—借書人」的多點傳遞模式
- **痛點**: 需要清楚了解保管責任和期限

### **4. 社區組織者 (Community Organizers)**
- **年齡**: 30-60歲
- **技術水平**: 中等到高級
- **動機**: 管理社區資源，促進居民互動，與 NGO 合作建立交收點，將「書的流通」與「人的流通」結合
- **痛點**: 缺乏有效的社區資源管理工具

## 📖 **核心 User Stories**

### **Epic 1: 書籍上傳和放置 (Book Upload & Placement)**

#### **Story 1.1: 書籍註冊和基本信息錄入**
**作為** 書籍擁有者  
**我想要** 輕鬆註冊我的書籍到系統中  
**以便** 讓社區成員知道我有什麼書可以分享

**用戶操作流程**:
1. 登入 DLLM 系統
2. 點擊「Create Item」按鈕
3. 進入「Create Item」表單頁面，填寫以下必填和選填字段：
   **必填欄位**: 書名、分類（逗號分隔）、狀態（NEW, GOOD, FAIR, POOR）、語言（EN, ZH, JA 等）、可用狀態（AVAILABLE, UNAVAILABLE）
   **選填欄位**: 描述、上傳圖片、出版年份（如 2024）
4. 點擊「CREATE ITEM」按鈕完成書籍註冊，或點擊「CANCEL」取消操作

**驗收標準**:
- [ ] 支持書籍名稱輸入
- [ ] 支持分類標籤輸入（逗號分隔）
- [ ] 支持書籍狀態選擇（NEW, GOOD, FAIR, POOR）
- [ ] 支持語言選擇
- [ ] 支持可用狀態設置
- [ ] 支持可選描述輸入
- [ ] 支持圖片上傳功能
- [ ] 支持出版年份輸入（4位數格式）
- [ ] 表單驗證和錯誤提示
- [ ] 響應式設計支持多種設備

**技術需求**:
- 表單驗證和數據存儲
- 圖像上傳和處理
- 下拉選單組件
- 響應式表單設計
- 數據庫存儲和檢索

#### **Story 1.2: 書籍位置和交收點設置**
**作為** 書籍擁有者  
**我想要** 設置書籍的存放位置和交收方式  
**以便** 借書人知道如何獲取書籍

**用戶操作流程**:
1. 在書籍註冊完成後，進入「位置設置」頁面
2. 選擇書籍存放方式：
   - **個人保管**: 書籍存放在自己家中
   - **社區交收點**: 存放在指定的社區交收點
   - **混合模式**: 根據情況靈活選擇
3. 如果選擇個人保管：
   - 輸入家庭地址（可選擇隱私保護）
   - 設置可交收時間（如工作日晚上、週末等）
   - 選擇交收方式（上門取書、約定地點等）
4. 如果選擇社區交收點：
   - 從地圖上選擇附近的交收點
   - 查看交收點的詳細信息（地址、開放時間、設施）
   - 確認交收點是否接受該類書籍
   - 選擇符合生活路線的交收點（如居住地與工作地之間）
5. 設置通知偏好：
   - 借閱請求通知
   - 交收提醒
   - 逾期提醒
6. 點擊「保存設置」完成

**驗收標準**:
- [ ] 支持多種存放方式選擇
- [ ] 地圖式交收點選擇
- [ ] 靈活的交收時間設置
- [ ] 隱私保護選項
- [ ] 完整的通知設置
- [ ] 支持生活路線優化的交收點選擇

**技術需求**:
- 地理編碼和地圖服務
- 地址隱私保護機制
- 通知系統配置

#### **Story 1.3: 書籍狀態管理和更新**
**作為** 書籍擁有者  
**我想要** 隨時更新書籍狀態和信息  
**以便** 保持書籍信息的準確性

**用戶操作流程**:
1. 在「我的書籍」頁面查看所有已註冊的書籍
2. 點擊特定書籍進入「編輯」模式
3. 可以更新的信息：
   - 書籍狀態（可用、借出、維護中、遺失等）
   - 借閱條件和期限
   - 位置和交收方式
   - 個人描述和標籤
4. 系統自動記錄變更歷史
5. 如果書籍狀態變更為「借出」，系統自動通知借書人
6. 點擊「保存更新」完成

**驗收標準**:
- [ ] 實時狀態更新
- [ ] 變更歷史記錄
- [ ] 自動通知機制
- [ ] 批量操作支持

**技術需求**:
- 狀態管理系統
- 變更追蹤機制
- 實時通知推送

### **Epic 2: 書籍借閱 (Book Borrowing)**

#### **Story 2.1: 書籍搜索和發現**
**作為** 借書人  
**我想要** 快速找到我需要的書籍  
**以便** 開始借閱流程

**用戶操作流程**:
1. 在首頁使用搜索功能：
   - 輸入關鍵詞（書名、作者、主題等）
   - 使用高級搜索選項（分類、語言、出版年份等）
   - 按地理位置篩選（距離、交通方式等）
2. 瀏覽搜索結果：
   - 查看書籍基本信息（標題、作者、封面、評分）
   - 查看可用性和位置信息
   - 查看借閱條件和期限
   - 查看書籍擁有者的評價
3. 使用篩選和排序功能：
   - 按距離排序
   - 按評分排序
   - 按可用性篩選
   - 按借閱條件篩選
4. 點擊感興趣的書籍查看詳細信息

**驗收標準**:
- [ ] 多種搜索方式支持
- [ ] 智能搜索建議
- [ ] 靈活的篩選和排序
- [ ] 搜索結果分頁顯示
- [ ] 搜索歷史記錄

**技術需求**:
- 全文搜索引擎
- 地理位置索引
- 智能推薦算法

#### **Story 2.2: 借閱申請和確認**
**作為** 借書人  
**我想要** 申請借閱書籍並獲得確認  
**以便** 開始使用書籍

**用戶操作流程**:
1. 在書籍詳細頁面點擊「申請借閱」按鈕
2. 系統顯示借閱確認信息：
   - 書籍基本信息
   - 借閱條件和期限
   - 交收方式和地點
   - 預計交收時間
3. 填寫借閱申請：
   - 選擇借閱期限（在允許範圍內）
   - 選擇交收方式（上門取書、約定地點、交收點等）
   - 添加個人留言（可選）
   - 確認借閱條款
4. 點擊「提交申請」
5. 系統自動發送申請通知給書籍擁有者
6. 等待擁有者確認（通常24小時內）
7. 收到確認通知後，系統顯示交收詳情

**驗收標準**:
- [ ] 清晰的借閱條件展示
- [ ] 靈活的交收方式選擇
- [ ] 自動通知機制
- [ ] 申請狀態追蹤
- [ ] 借閱條款確認

**技術需求**:
- 申請流程管理
- 通知系統
- 狀態追蹤機制

#### **Story 2.3: 書籍交收和借閱開始**
**作為** 借書人  
**我想要** 順利完成書籍交收並開始借閱  
**以便** 使用書籍進行學習

**用戶操作流程**:
1. 收到借閱確認後，查看交收詳情：
   - 交收地點和時間
   - 聯繫方式
   - 交收注意事項
2. 按約定時間和地點進行交收：
   - 如果是上門取書：按地址前往
   - 如果是約定地點：按約定前往
   - 如果是交收點：按開放時間前往
3. 完成交收後：
   - 在系統中點擊「確認收到」
   - 系統記錄借閱開始時間
   - 開始計算借閱期限
4. 系統自動設置提醒：
   - 借閱到期提醒
   - 續借選項提醒
   - 交還提醒

**驗收標準**:
- [ ] 清晰的交收指引
- [ ] 交收確認機制
- [ ] 自動期限計算
- [ ] 智能提醒系統

**技術需求**:
- 交收流程管理
- 時間計算引擎
- 提醒系統

#### **Story 2.4: 連鎖借閱 (Chain Borrowing)**
**作為** 借書人  
**我想要** 通過連鎖借閱模式借到不在附近的書籍  
**以便** 擴大可借閱的書籍範圍

**用戶操作流程**:
1. 搜索書籍時發現想要的書籍不在附近
2. 系統顯示「連鎖借閱」選項
3. 查看借閱鏈路：
   - 書籍當前持有人位置
   - 可用的交收點
   - 預計傳遞時間
   - 參與傳遞的人員
4. 選擇連鎖借閱：
   - 選擇最近的交收點
   - 確認傳遞路徑
   - 了解傳遞費用（如有）
5. 提交連鎖借閱申請
6. 系統協調傳遞過程：
   - 通知相關人員
   - 安排傳遞時間
   - 追蹤傳遞狀態
7. 在指定交收點接收書籍

**驗收標準**:
- [ ] 清晰的借閱鏈路展示
- [ ] 智能路徑規劃
- [ ] 自動協調機制
- [ ] 傳遞狀態追蹤
- [ ] 支持「書主—持有人—借書人」的多點傳遞模式

**技術需求**:
- 圖數據庫實現借閱鏈路
- 路徑規劃算法
- 協調和通知系統

### **Epic 3: 借閱管理 (Borrowing Management)**

#### **Story 3.1: 借閱狀態追蹤**
**作為** 借書人  
**我想要** 隨時查看我的借閱狀態  
**以便** 管理我的借閱活動

**用戶操作流程**:
1. 在「我的借閱」頁面查看所有借閱記錄
2. 查看當前借閱的書籍：
   - 借閱開始時間
   - 預計歸還時間
   - 剩餘借閱天數
   - 書籍狀態和位置
3. 查看借閱歷史：
   - 已歸還的書籍
   - 借閱評價和評論
   - 借閱統計數據
4. 使用篩選和搜索功能：
   - 按狀態篩選（當前借閱、逾期、已歸還）
   - 按時間篩選
   - 按書籍類型篩選
5. 設置借閱提醒：
   - 到期提醒時間
   - 提醒方式（郵件、短信、推送）

**驗收標準**:
- [ ] 實時狀態更新
- [ ] 完整的借閱歷史
- [ ] 靈活的篩選功能
- [ ] 個性化提醒設置

**技術需求**:
- 借閱狀態管理
- 歷史記錄系統
- 提醒和通知系統

#### **Story 3.2: 續借和歸還**
**作為** 借書人  
**我想要** 續借書籍或按時歸還  
**以便** 延長使用時間或完成借閱

**用戶操作流程**:
1. **續借流程**:
   - 在借閱到期前收到續借提醒
   - 點擊「申請續借」按鈕
   - 選擇續借期限（在允許範圍內）
   - 添加續借理由（可選）
   - 提交續借申請
   - 等待擁有者確認
   - 收到確認後延長借閱期限

2. **歸還流程**:
   - 在借閱到期前收到歸還提醒
   - 選擇歸還方式：
     - 親自歸還給擁有者
     - 通過交收點歸還
     - 約定地點歸還
   - 按約定完成歸還
   - 在系統中點擊「確認歸還」
   - 系統記錄歸還時間
   - 可以對書籍和借閱體驗進行評價

**驗收標準**:
- [ ] 簡單的續借申請流程
- [ ] 靈活的歸還方式選擇
- [ ] 自動提醒機制
- [ ] 評價和反饋系統

**技術需求**:
- 續借流程管理
- 歸還流程管理
- 評價和反饋系統
